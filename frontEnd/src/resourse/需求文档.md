# Heaven Films 电影票在线预售系统需求文档
**版本：** V2.0  
**定位：** 极简、高交互的 C 端电影购票平台

---

## 1. 项目概述

### 1.1. 项目背景
随着数字化生活普及，用户对购票体验的要求从“能买到”转向“好用、好看”。本项目旨在开发一个基于 B/S 架构的**在线电影票预售系统**，通过 Vue 3 + SpringBoot 技术栈，为用户提供沉浸式的选座购票体验，简化传统复杂的购票流程。

### 1.2. 项目目标
*   **交互体验极致化**：实现平滑的电影切换、动态的选座交互以及响应式的页面布局。
*   **购票流程闭环**：涵盖电影浏览、影院选择、实时选座、模拟下单及个人行程管理。
*   **性能与稳定性**：利用虚拟滚动处理大量电影数据，确保在高并发选座场景下的前端流畅度。

---

## 2. 用户角色

| 角色 | 描述 | 核心职责 |
| :--- | :--- | :--- |
| **普通用户** | 系统的核心使用者 | 浏览电影详情、关注动态排片、在线选座购票、管理个人收藏与订单。 |

---

## 3. 功能性需求

### 3.1. 核心功能模块

| 功能模块 | 子功能 | 具体需求描述 |
| :--- | :--- | :--- |
| **用户认证** | 注册与登录 | 支持手机号/账号注册，提供安全的登录验证机制，支持个人头像上传与资料维护。 |
| **电影探索** | 多维度筛选 | 首页展示“正在热映”与“即将上映”精选（各10部），支持按剧情、动作、喜剧等类型实时过滤。 |
| | 排行榜单 | 根据用户评分自动生成实时电影排名，引导用户发现热门好片。 |
| **购票核心** | 影院列表 | 列表式展示合作影院，支持查看影院地址、特色影厅标签（如 IMAX, 4DX）。 |
| | 实时选座 | **图形化选座界面**：动态展示座位图，实时反馈已选、可选、已售状态。单个订单限购 5 张。 |
| | 模拟下单 | 选座确认后生成唯一订单编号，支持模拟支付流程并更新座位占用状态。 |
| **个人中心** | 订单管理 | 记录用户所有购票行程，支持查看订单详情，包含放映时间、影院地点、座位号等。 |
| | 关注收藏 | 用户可收藏心仪电影，方便在“我的收藏”中快速查看预售动态。 |

---

## 4. 非功能性需求

*   **性能要求**：
    *   电影列表页支持虚拟滚动/无限加载，确保海量电影数据下页面无卡顿。
    *   页面切换采用平滑过渡动效，提升产品高级感。
*   **交互规范**：
    *   所有操作按钮（如购票、返回顶部）需具备明确的悬停与点击反馈。
    *   选座界面需具备防重复点击与实时校验逻辑。
*   **安全性**：
    *   用户敏感数据（如密码）需进行加密存储。
    *   通过 JWT 等机制保护下单接口，防止非法越权操作。

---

## 5. 数据实体（核心）

*   **用户 (User)**：ID、账号、密码、昵称、头像、联系方式。
*   **电影 (Movie)**：ID、标题、海报、导演、主演、类型、时长、上映日期、评分。
*   **影院 (Cinema)**：ID、影院名称、详细地址、服务标签（IMAX/退票等）。
*   **场次 (Schedule)**：ID、关联电影、关联影厅、放映开始/结束时间、票价。
*   **订单 (Order)**：ID、唯一编号、关联用户、关联场次、已选座位号、总金额、状态（已完成/待支付）。
*   **收藏 (Favorite)**：关联用户 ID、关联电影 ID。

---

**备注：** 本系统为纯前端展示与用户业务逻辑演示系统，不包含后台管理逻辑，旨在通过极致的 C 端交互体现系统设计的专业性。
