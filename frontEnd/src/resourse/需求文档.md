# Heaven Films 电影票在线预售系统 - 资深工程师重构方案 (V3.0)

**重构目标：** 打造一个具备生产级质量、逻辑严密、视觉顶级的电影票预售系统。

---

## 一、 核心架构重构 (Engineering Architecture)

### 1.1 数据库与实体层对齐
*   **统一标识符**：全表 `id` 统一使用 `Long` (BIGINT)，确保大容量扩展性。
*   **消除冗余**：移除 `User` 实体中冗余的 `selected` 字段，全面切换为 `favorites` 关联表。
*   **精确映射**：修正 `Movie` 实体中的字段名（如 `actors` 替代 `cast`），增加 `is_showing` 状态位。

### 1.2 后端逻辑强化
*   **统一响应协议**：引入标准的 `Result<T>` 包装类，包含 `code`, `message`, `data`。
*   **异常拦截**：建立 `GlobalExceptionHandler`，优雅处理业务逻辑冲突（如：座位已被占用）。
*   **选座并发处理**：
    *   在创建订单时锁定 `booked_seats`。
    *   利用数据库唯一索引 `uk_sch_seat` (schedule_id, row, col) 物理级防止重选。

### 1.3 前端工程化重构
*   **严格类型体系**：完善 `src/types/index.ts`，消除 `any`，确保 API 请求与响应的类型安全。
*   **模块化状态管理**：重构 Store 逻辑，使用 Pinia/Vuex 规范化存储用户信息与地理位置。
*   **高性能组件**：对电影卡片、选座图进行性能优化，减少不必要的重绘。

---

## 二、 视觉与交互方案 (UI/UX Design)

### 2.1 视觉风格：极简主义 (Minimalist Cinematic)
*   **配色方案**：
    *   背景：`#FFFFFF` (纯净白) 或 `#0F0F0F` (沉浸黑)。
    *   主色：`#FF3D00` (动感橙) 用于购票与高亮。
    *   中性色：`#F5F5F7` 用于背景底色，`#1D1D1F` 用于文字。
*   **字体规范**：优先使用系统默认字体栈（PingFang SC, -apple-system），强调行间距 (1.6) 以提升阅读舒适度。

### 2.2 核心交互链路
1.  **沉浸式首页**：采用渐变式 Hero Carousel 展示头部大片，横向滚动展示分类电影。
2.  **动态详情页**：海报背景高斯模糊效果，突出电影核心信息。
3.  **极简选座交互**：
    *   直观的座位布局图，支持缩放与多选。
    *   右侧实时计算总价并显示所选座位标签。
